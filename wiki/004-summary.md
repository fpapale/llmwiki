**Summary – Supervised Learning & Classification (SP4TE Lab, 2024‑2025)**  

| Topic | Key Points |
|-------|------------|
| **Logistic Regression** | • Probabilistic linear model for binary classification.<br>• Training: estimate weight vector **W** and bias **b** to maximise likelihood (or minimise cross‑entropy).<br>• Uses linear combination *z = W·x + b* → sigmoid → class probability.<br>• Decision threshold (commonly 0.5) converts probability to binary output. |
| **Performance Metrics** | • **Accuracy** = (TP + TN) / (TP + TN + FP + FN) – useful only on balanced, high‑quality data.<br>• **Precision** = TP / (TP + FP) – correctness of positive predictions.<br>• **Recall (Sensitivity)** = TP / (TP + FN) – ability to capture actual positives.<br>• **F1‑Score** = 2·(Precision·Recall)/(Precision + Recall) – harmonic mean, balances precision & recall.<br>• **ROC Curve** (TPR vs. FPR) and **AUC** – evaluate trade‑off across thresholds; larger AUC → better model. |
| **Support Vector Machines (SVM)** | • Finds hyperplane with maximum margin for linearly separable data.<br>• For non‑linear cases, maps data to higher‑dimensional space via **kernel trick** (e.g., polynomial, RBF) without explicit transformation.<br>• Soft‑margin optimisation handles overlapping classes. |
| **Decision Trees** | • Hierarchical, rule‑based model: each node tests an attribute, branches to children, leaves give class (or regression value).<br>• Splitting criteria: **Information Gain** (entropy reduction) or **Gini impurity**.<br>• Advantages: interpretability, simple implementation.<br>• Disadvantages: high variance, limited expressive power for complex functions. |
| **Ensemble Methods** | **Bagging / Random Forest**<br>• Build many trees on bootstrap samples; each split uses a random subset of features → decorrelated trees.<br>• Prediction: majority vote (classification) or average (regression).<br>• Pros: reduced over‑fitting, handles missing data.<br>• Cons: slower, higher memory, less interpretable than a single tree.<br><br>**Boosting**<br>• Train models sequentially; each focuses on errors of the previous one (weights ↑ for mis‑classified instances).<br>• Combine weak learners (e.g., shallow trees) into a strong predictor (e.g., AdaBoost, Gradient Boosting). |
| **Key Take‑aways** | • Choose the metric that matches the problem context (e.g., precision for rare‑event detection, recall for medical screening).<br>• Linear models (logistic regression) are fast and interpretable; kernel SVMs handle non‑linear boundaries; tree‑based ensembles often give the best practical performance on heterogeneous data.<br>• Proper model selection, hyper‑parameter tuning, and validation (e.g., out‑of‑bag error for Random Forest) are essential to avoid over‑fitting. |
